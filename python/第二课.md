# 第 2 课：L4 工具调用与工具边界 — 课程记录

## 本课目标

把工具从「能调用」升级到输入清晰、边界稳定、可测试。

## 核心链路

```
模型想调用工具
  ↓
Pydantic schema 校验参数  ← 硬约束：类型、范围、extra="forbid"
  ↓ 通过            ↓ 不通过
@tool 执行          → 返回校验错误（工具不执行）
  ↓
纯业务函数 _xxx()  ← 可独立测试（不依赖 LangChain）
  ↓
返回结果给模型
```

## 关键修正记录

### 修正 1：`Literal` 的来源

- ❌ 以为是 pydantic 库
- ✅ 来自 Python 标准库 `typing`，仅类型注解，**运行时无强制约束力**
- `@tool` 把类型信息传给模型，但模型不遵守时 Python 不会阻止
- 所以需要 `args_schema=Pydantic` 做运行时校验

### 修正 2：模型能看到工具哪些信息

- ❌ "模型完全凭借工具描述进行调用"
- ✅ 模型能看到：**函数名 + 参数名 + 参数类型 + docstring + description** 全部信息
- 但看不到 Python 代码实现逻辑

### 修正 3：`Decimal` 是定点十进制，不是「十进制类型」

- `float`: `0.1 + 0.2 = 0.30000000000000004`（二进制浮点无法精确表示 0.1）
- `Decimal`: `Decimal("0.1") + Decimal("0.2") = Decimal('0.30')`（精确）
- 金融计算（月供×360期）必须用 `Decimal`，否则累积误差可见

## 三层模式详解（本课核心）

```python
# 第一层：Pydantic schema — 参数校验（硬约束）
class LoanMonthlyPaymentInput(BaseModel):
    model_config = ConfigDict(extra="forbid")
    principal: Decimal = Field(gt=0, description="本金")
    annual_rate: Decimal = Field(gt=0, le=100, description="年化利率，百分数")
    months: PositiveInt = Field(description="期数")

# 第二层：纯业务函数（可独立测试）
def _loan_monthly_payment(principal, annual_rate, months) -> Decimal:
    rate = annual_rate / 1200
    x = (1 + rate) ** months
    money = (principal * rate * x) / (x - 1)
    return money.quantize(Decimal("0.01"), rounding=ROUND_HALF_UP)

# 第三层：@tool 包装（薄薄一层）
@tool(args_schema=LoanMonthlyPaymentInput)
def loan_monthly_payment(principal, annual_rate, months) -> Decimal:
    return _loan_monthly_payment(principal, annual_rate, months)
```

### 为什么拆三层？

| 层 | 职责 | 可独立测试？ | 依赖 LangChain？ |
|---|---|---|---|
| Pydantic schema | 参数类型/范围/额外字段拦截 | ✅ Pytest | ❌ |
| 纯业务函数 `_xxx` | 实际计算逻辑 | ✅ Pytest | ❌ |
| `@tool` 包装 | 暴露给模型 | ⚠️ 需 agent 一起测 | ✅ |

### 与你之前项目的对应关系

```
你的 SQL agent：
  模型生成 SQL → 关键词检测（你的校验层）→ 执行 SQL（执行层）

L4 三层模式：
  模型传参 → Pydantic schema（L4 的校验层）→ 业务函数（执行层）

区别：
  - 你的校验：字符串关键词匹配（有绕过风险）
  - L4 校验：类型 + 范围 + extra 拦截（更结构化）
```

## Field 约束速查

| 约束 | 含义 | 示例 |
|---|---|---|
| `gt=0` | greater than，大于 0 | `principal: Decimal = Field(gt=0)` |
| `ge=0` | greater or equal | |
| `lt=100` | less than | |
| `le=100` | less or equal | `annual_rate: Decimal = Field(le=100)` |
| `PositiveInt` | 正整数（>0 的 int） | `months: PositiveInt` |
| `FiniteFloat` | 有限浮点数（非 inf/NaN） | |
| `extra="forbid"` | 禁止未定义的字段 | `model_config = ConfigDict(extra="forbid")` |
| `description` | 字段说明（模型能看到） | `Field(description="本金")` |

## Pydantic 校验错误示例（来自 Cell 22 输出）

```
principal=0 → "principal: Input should be greater than 0"
annual_rate=-1 → "annual_rate: Input should be greater than 0"
a="abc" → "a: Input should be a valid number, unable to parse string as a number"
```

这些错误在工具执行前就被 Pydantic 拦截，**不会进入业务函数**。

## 术语速查

| 术语 | 含义 |
|---|---|
| `Literal` | typing 的类型注解，限制参数可选值（运行时无强制力） |
| `parse_docstring=True` | 让 `@tool` 解析 Google 风格 docstring 中的 Args/Returns/Raises |
| `args_schema` | `@tool` 参数，指定 Pydantic model 做运行时参数校验 |
| `ConfigDict(extra="forbid")` | Pydantic 配置，禁止传递 schema 未定义的字段 |
| `Decimal` | 定点十进制数，用于金融等需要精确计算的场景 |
| `quantize("0.01")` | Decimal 的四舍五入到指定精度 |
| 三层模式 | Pydantic schema → 业务函数 → @tool 包装的架构 |
