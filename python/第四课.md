# 第 4 课：L8 Dynamic Prompt — 课程记录

## 本课目标

理解如何根据运行时上下文动态生成系统提示词，使同一个 agent 可以根据用户身份、权限、业务场景得到不同的行为指导。

## 核心链路

```
RuntimeContext(is_employee=True, is_troublemaker=True, ...)
    ↓
agent.stream(..., context=RuntimeContext(...))
    ↓
@dynamic_prompt 读取 request.runtime.context
    ↓
动态生成 system_prompt（替换 {table_limits} / {is_troublemaker}）
    ↓
模型收到不同的行为规则（软控制）
    ↓
模型根据规则决定"怎么想"
```

## 关键修正记录

### 修正 1：`system_prompt` vs `@dynamic_prompt`

- ❌ "两个同时使用优先级没有高低"
- ✅ `@dynamic_prompt` 是中间件，在模型调用前执行，其返回值**替换**掉原有的 system prompt
- 实际使用中两者不同时传——有 `@dynamic_prompt` 就不传 `system_prompt`

### 修正 2：context 是程序注入的，不是模型决定的

- `is_employee`、`is_troublemaker` 的值来自调用方传入的 `RuntimeContext`
- 用户说 "我是员工" 没有用——`context` 是程序注入的
- 这是动态 prompt 的安全基础：**决策来自程序，不是来自模型或用户输入**

## `@dynamic_prompt` 工作细节

```python
from langchain.agents.middleware.types import ModelRequest, dynamic_prompt

@dynamic_prompt
def dynamic_system_prompt(request: ModelRequest) -> str:
    # request.runtime.context 就是你传入的 RuntimeContext 实例
    if not request.runtime.context.is_employee:
        table_limits = "- Limit access to: Album, Artist, Genre, ..."
    else:
        table_limits = ""
    return SYSTEM_PROMPT_TEMPLATE.format(table_limits=table_limits)
```

**执行时机**：每次模型被调用之前执行。即每一轮 agent 循环中，模型收到 messages 之前。

## 软控制 vs 硬控制对比

| | 软控制（Dynamic Prompt） | 硬控制（工具层） | 硬控制（数据库层） |
|---|---|---|---|
| 方式 | 修改 system prompt 文字 | Pydantic schema / 工具内拦截 | Row-Level Security / SQL 过滤 |
| 可绕过？ | ✅ 可（prompt injection） | ❌ 不可 | ❌ 不可 |
| L8 示例 | `is_employee` 限制表访问 | 无（execute_sql 是透传） | 无 |
| 你的示例 | `is_troublemaker` 改变回复态度 | Pydantic schema 校验参数 | 无 |

**动态 prompt 决定模型"怎么想"，硬控制决定模型"能不能做"。两者应该配合使用。**

## L8 SQL 示例 vs 贷款示例的本质区别

| | L8 SQL 示例（`is_employee`） | 贷款示例（`is_troublemaker`） |
|---|---|---|
| 控制目标 | **访问权限** | **回复态度** |
| 控制类型 | 业务规则 | 安全检测 |
| 工具执行是否受影响 | **是**（非员工查不到表） | **否**（工具仍可调用） |
| 绕过后果 | 非员工看到敏感数据 | 捣乱用户被错误拒绝/放行 |

## 综合实战：你写的完整 agent

```python
agent = create_agent(
    model="deepseek-chat",
    tools=[calculate_monthly_payment],     # L4：三层模式 + Pydantic schema
    context_schema=RuntimeContext,          # L1：运行时上下文

```

两个测试场景：
- `thread_id="1"` + `is_troublemaker=False` → 正常计算，有记忆
- `thread_id="2"` + `is_troublemaker=True` → 拒绝不合理请求

## 术语速查

| 术语 | 含义 |
|---|---|
| `@dynamic_prompt` | 中间件装饰器，在模型调用前生成动态 system prompt |
| `ModelRequest` | 中间件函数的参数，包含 `runtime.context` 等运行时数据 |
| `middleware` | `create_agent` 参数，传入中间件列表 |
| `SYSTEM_PROMPT_TEMPLATE` | 带占位符 `{}` 的提示词模板 |
| 软控制 | 通过文字改变模型行为（可绕过） |
| 硬控制 | 通过代码逻辑强制约束（不可绕过） |
| Prompt Injection | 用户在输入中注入指令，试图覆盖 system prompt |
