# 第 5 课：L9 Human-in-the-Loop — 课程记录

## 本课目标

掌握高风险工具执行前的人类审批流程——中断-审批-恢复的状态机。

## 核心链路

```
用户提问
  ↓
模型提出工具调用
  ↓
HumanInTheLoopMiddleware 拦截
  ↓
返回 __interrupt__（暂停执行）
  ├─ approve → Command(resume={"decisions": [{"type": "approve"}]})
  │    ↓
  │  工具执行 → 可能再次 interrupt → 循环直到完成
  │    ↓
  └─ reject → Command(resume={"decisions": [{"type": "reject", "message": "..."}]})
       ↓
      模型收到 reject 消息，生成回复
```

## 关键代码

```python
from langchain.agents.middleware import HumanInTheLoopMiddleware

agent = create_agent(
    ...,
    checkpointer=InMemorySaver(),          # ← 必须：中断恢复
    middleware=[
        HumanInTheLoopMiddleware(
            interrupt_on={
                "execute_sql": {
                    "allowed_decisions": ["approve", "reject"]
                }
            },
        ),
    ],
)

# 中断后恢复
result = agent.invoke(
    Command(resume={"decisions": [{"type": "approve"}]}),
    config={"configurable": {"thread_id": "1"}},  # ← 同一个 thread_id
    context=RuntimeContext(db=db),
)
```

## 关键修正记录

### 修正 1：HITL 拦截的是工具动作，不是整轮对话

- 一次用户问题可能触发多次工具调用
- 每次工具调用前都会 interrupt，每次都需要 approve/reject
- `while __interrupt__ in result` 循环处理这种情况

### 修正 2：L6 的 checkpoint vs L9 的 checkpoint

| | L6（记忆） | L9（HITL） |
|---|---|---|
| 保存内容 | messages + 状态 | messages + 状态 + **执行暂停点** |
| 恢复方式 | 下一轮直接提问 | `Command(resume=...)` 从中断点继续 |
| 中断机制 | 无 | `__interrupt__` 暂停 graph 执行 |

两者使用同一套 checkpointer 机制，但 L9 多了 **位置恢复** 的需求。

### 修正 3：`__interrupt__` 的数据结构

当 HITL 拦截时，result 会包含：

```python
result["__interrupt__"][-1].value["action_requests"][-1]
# → {"description": "Tool: execute_sql\nArgs: {'query': '...'}", ...}
```

读取这些信息可以拼接审批界面。

## 三个必须同时存在的要素

1. **`checkpointer`** — 没有它，中断后状态丢失
2. **`interrupt_on={"tool_name": ...}`** — 声明哪些工具需要审批
3. **同一个 `thread_id`** — 恢复时定位到暂停点

## 常见 UX 方案：Trust for Session

```
🛑 Agent 想要执行 SQL: SELECT ...
[Approve] [Approve for all] [Reject]
```

- 单次 approve：仅本次放行
- Approve all：当前 thread_id 后续相同工具不再中断
- Reject + message：模型收到拒绝理由

## HITL 适用场景

- 执行 SQL（INSERT/UPDATE/DELETE）
- 发送邮件
- 执行支付
- 删除文件
- 部署操作
- 任何不可逆或高风险操作

## 术语速查

| 术语 | 含义 |
|---|---|
| `HumanInTheLoopMiddleware` | 拦截工具调用的中间件 |
| `interrupt_on` | 声明哪些工具需要审批的配置 |
| `allowed_decisions` | 审批可选决策列表 |
| `__interrupt__` | 中断信号，result 中包含该 key 表示暂停 |
| `Command(resume=...)` | 从中断点恢复执行的命令 |
| action request | HITL 拦截的最小单位：一次工具调用的请求 |
| Trust for Session | 一次性批准整轮对话中所有工具调用的 UX 模式 |
